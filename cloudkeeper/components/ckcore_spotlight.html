Spotlight: API

The API of ckcore is exposed via http and websocket. You can access it via http://<cloudkeeper-address>:8900/api-doc

ckcore has two API endpoints to connect to for CLI purposes:
http://<cloudkeeper-address>:8900/cli/evaluate
http://<cloudkeeper-address>:8900/cli/execute

cli/evaluate functinality is used internally on every cli/execute before the command execution.

Here is a simulation of sending a cksh query to the CLI API. We will evaluate the query before executing it for demonstration. Also we introduce this query with a typo to show the response if not successful.

Evaluate

Evaluate, correct: match is("resource") limit 1
$ echo 'graph=ck match is("resource") limit 1' | http :8900/cli/evaluate

Content-Length: 47
Content-Type: application/json; charset=utf-8
Date: Wed, 06 Oct 2021 15:13:08 GMT
Server: Python/3.9 aiohttp/3.7.4.post0

[
    {
        "execute_query": "is(\"resource\") limit 1"
    }
]

ckcore is the central HUB for everything Cloudkeeper does. You can discover ckcore APIs directly via WebBrowser (exposed at http://<cloudkeeper-address>:8900/) or in our repository

There will be examples of typical API Calls in the in depth descriptions of every Cloudkeeper component.
